# v3 — Multi-User (2026-04-27)

## Design Decisions

- Email/password only (no OAuth). JWT stateless. Invite-only registration.
- Admin seeded via CLI. SMTP for password reset. Clean slate (no v2 widget migration).
- Access token (30 min) in memory, refresh token (7 days) as `httpOnly` cookie.

## Phase 1: Backend Auth Foundation

- Models: `User`, `InviteCode`, `RefreshToken`. Nullable `user_id` FK on `Widget`.
- Auth: `bcrypt` hashing (passlib dropped), `python-jose` JWT, SHA-256 refresh tokens.
- Endpoints: register, login, refresh, logout, `GET /auth/me`.
- CLI: `create-admin`. Migration `033d10d1c2d0`.
- Deps: `python-jose`, `bcrypt`, `python-multipart`, `email-validator`.

## Phase 2: Password Reset

- `PasswordResetToken` model. Migration `c5336e6a41d5`.
- `aiosmtplib` email service (falls back to console logging).
- Endpoints: `POST /auth/forgot-password`, `POST /auth/reset-password`.
- Config: `SMTP_*`, `FRONTEND_URL`, `RESET_TOKEN_EXPIRE_MINUTES`.

## Phase 3: User-Scoped Widgets + Admin API

- Migration `02aa7e61713a`: `widgets.user_id` NOT NULL.
- All widget CRUD filtered by `current_user.id`.
- Default widgets on registration: NY time + AAPL, MSFT, BTC-USD.
- Admin router: invite code CRUD + user list with widget counts.

## Phase 4: Frontend — Routing + Landing + Auth

- `react-router-dom`. Routes: `/` (landing), `/login`, `/register`, `/dashboard`.
- `AuthContext`: token-in-memory, cookie refresh, axios interceptor (401 → refresh).
- `ProtectedRoute` + `AdminRoute` guards. Landing page with hero + features.
- `apiClient` with `withCredentials: true`.

## Phase 5: Frontend — Password Reset + Admin Panel

- `ForgotPasswordPage`, `ResetPasswordPage`, `AdminPage`.
- Admin: invite code CRUD + read-only user list.
- `queryClient.clear()` on login/register/logout (cross-user data leakage fix).
- Backend: `ensure_default_assets()` on startup, `PROTECTED_ASSET_IDS`.

## Phase 6: Polish

- Version bump `2.0.0` → `3.0.0`. Scroll fix, TS build fix (`react-grid-layout` v2).
- Auth session expiry → redirect to `/login`. 404 catch-all. Responsive landing.

### State at v3.0.0
- 7 tables: `users`, `invite_codes`, `refresh_tokens`, `password_reset_tokens`, `assets`, `price_snapshots`, `widgets`.
- Full auth flow, admin panel, user-scoped dashboard.
