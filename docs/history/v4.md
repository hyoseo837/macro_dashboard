# v4 ‚Äî News Widgets (2026-04-29)

Two-phase approach: Phase A (RSS headlines, no AI) first, Phase B (Gemini) deferred.

## Planning

- Single-feed per widget (like assets). Predefined feed catalog (22 feeds, not user-editable).
- On-demand feed activation. 7-day article retention. Minimum 2x1 widget size. Hourly refresh.

## Phase 1: Backend ‚Äî Models + Feed Catalog + RSS Service

- **Models**: `NewsFeed` (feed_key, source_name, topic, country, feed_url, last_fetched_at) and `NewsArticle` (feed_id FK, title, url, source_name, published_at, fetched_at). `WidgetType.news` added.
- **Migration**: `0899255d7d07` ‚Äî tables + `'news'` enum value.
- **Feed catalog** (`app/services/news.py`): 22 feeds ‚Äî BBC (6), CNN (5), Reuters (3), NYT (5), HN (1), Korea Herald (1), CBC (1).
- **RSS service**: `feedparser` + `httpx`. `activate_feed` / `deactivate_orphan_feeds` / `fetch_feed` / `refresh_all_feeds` / `cleanup_old_articles`.
- **Widget router**: news validation, background fetch after commit, orphan cleanup on delete.
- **Bug fix**: `asyncio.create_task` moved after `db.commit()` to avoid race condition.
- **Dep**: `feedparser ^6.0.12`.

## Phase 2: Backend ‚Äî API + Scheduler

- `GET /news/catalog` ‚Äî all 22 feeds from catalog (public).
- `GET /news/articles?feed_id=X&limit=N` ‚Äî cached articles, ordered by `published_at` desc.
- Scheduler: hourly `refresh_all_feeds`, daily `cleanup_old_articles` at 3 AM, both run on startup.

## Phase 3: Frontend ‚Äî News Widget Component

- Types: `NewsWidgetConfig`, `NewsFeedCatalogEntry`, `NewsArticle`. `WidgetType` includes `'news'`.
- `useNewsCatalog` (stale: infinity), `useNewsArticles` (stale: 5 min).
- `NewsWidget`: headline list ‚Äî title (link, new tab) + source badge + relative time ("2h", "3d"). Count scales with size (4 at 2x1, 10 at 2x2, 15 at 2x3+).
- CSS: `.news-widget-*` with scrollable list, hover states, thin scrollbar.

## Phase 4: Frontend ‚Äî Add/Edit News Widget

- Type picker: Newspaper icon. Feed picker: searchable list from `/news/catalog`.
- Default size 2x1, placed at first free position. Label auto-fills, user-editable.
- Edit modal: label only (feed_id immutable).
- Caddyfile: added `/news/*` to reverse proxy.

## Phase 5: Documentation Restructuring

- Split `HISTORY.md`, `PLAN_future.md`, `API.md` into focused sub-files (<150 lines each) under `docs/{history,plan,api}/`.
- Top-level files replaced with index tables containing a "When to read" column ‚Äî keyword hints so agents only load relevant files.
- Added "Documentation conventions" section to `CLAUDE.md` defining the method: index-first navigation, keyword hints, separation of concerns (CLAUDE.md = current state, plan = WHY, history = WHAT/WHEN, api = HTTP contract).

### State at v4 (Phase A)
- 9 tables: added `news_feeds`, `news_articles`.

## Phase B: AI-Powered News (Gemini) ‚Äî 2026-04-30

### Phase B1: Foundation
- **Models**: `ArticleCluster` table (label, summary, topic, article_count, expires_at). `NewsArticle` gains `cluster_id` (FK) and `duplicate_of_id` (self-FK).
- **Migrations**: `ee3945059e8c` (initial AI columns), `64c6ba48a9aa` (move summary to cluster, drop per-article summary).
- **Config**: `GEMINI_API_KEY` in `Settings` (empty = AI disabled).
- **AI service** (`app/services/ai.py`): `cluster_articles` groups headlines by event and generates short one-liner summaries. Uses `google-genai` (Gemini 2.5 Flash). Rate-limited at 4s between calls.
- **Dep**: `google-genai ^1.74.0`, httpx bumped to `^0.28.0`.

### Phase B2: Widget Modes + API
- **Widget modes**: `config.mode` discriminant ‚Äî `single` (default), `topic`, `overall`.
- **API**: `GET /news/ai-status`, `GET /news/topics`, `GET /news/articles/topic?topic=X`, `GET /news/articles/clustered`. Topic and clustered endpoints both return `ClusteredArticleSchema[]`.
- **Widget validation**: topic mode activates all feeds for that topic, overall activates all feeds.
- **Orphan cleanup**: updated to respect topic/overall widgets.
- **Scheduler**: `scheduled_ai_pipeline` runs every 65 min, 5-min startup delay.

### Phase B3: Briefing UI
- **Frontend**: topic/overall widgets show briefing format ‚Äî event one-liner + source badges (BBC, CNN, NYT) linking to original articles. Single-feed widgets unchanged (headline list).
- **AddWidgetModal**: mode selector (Single/Topic/Overall), topic picker.
- No per-article summaries ‚Äî only cluster-level one-liners.

### State at v4B
- 10 tables: added `article_clusters` (with `summary` column).
- 2 new columns on `news_articles`: `cluster_id`, `duplicate_of_id`.
- 3 news widget modes: single (headline list), topic (briefing), overall (briefing).
