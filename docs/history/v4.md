# v4 — News Widgets

Two-phase approach: Phase A (RSS headlines, no AI) first, Phase B (Gemini) deferred.

## Planning (2026-04-29)

- Single-feed per widget (like assets). Predefined feed catalog (22 feeds, not user-editable).
- On-demand feed activation. 7-day article retention. Minimum 2x1 widget size. Hourly refresh.

## Phase 1: Backend — Models + Feed Catalog + RSS Service (2026-04-29)

- **Models**: `NewsFeed` (feed_key, source_name, topic, country, feed_url, last_fetched_at) and `NewsArticle` (feed_id FK, title, url, source_name, published_at, fetched_at). `WidgetType.news` added.
- **Migration**: `0899255d7d07` — tables + `'news'` enum value.
- **Feed catalog** (`app/services/news.py`): 22 feeds — BBC (6), CNN (5), Reuters (3), NYT (5), HN (1), Korea Herald (1), CBC (1).
- **RSS service**: `feedparser` + `httpx`. `activate_feed` / `deactivate_orphan_feeds` / `fetch_feed` / `refresh_all_feeds` / `cleanup_old_articles`.
- **Widget router**: news validation, background fetch after commit, orphan cleanup on delete.
- **Bug fix**: `asyncio.create_task` moved after `db.commit()` to avoid race condition.
- **Dep**: `feedparser ^6.0.12`.

## Phase 2: Backend — API + Scheduler (2026-04-29)

- `GET /news/catalog` — all 22 feeds from catalog (public).
- `GET /news/articles?feed_id=X&limit=N` — cached articles, ordered by `published_at` desc.
- Scheduler: hourly `refresh_all_feeds`, daily `cleanup_old_articles` at 3 AM, both run on startup.

## Phase 3: Frontend — News Widget Component (2026-04-29)

- Types: `NewsWidgetConfig`, `NewsFeedCatalogEntry`, `NewsArticle`. `WidgetType` includes `'news'`.
- `useNewsCatalog` (stale: infinity), `useNewsArticles` (stale: 5 min).
- `NewsWidget`: headline list — title (link, new tab) + source badge + relative time ("2h", "3d"). Count scales with size (4 at 2x1, 10 at 2x2, 15 at 2x3+).
- CSS: `.news-widget-*` with scrollable list, hover states, thin scrollbar.

## Phase 4: Frontend — Add/Edit News Widget (2026-04-29)

- Type picker: Newspaper icon. Feed picker: searchable list from `/news/catalog`.
- Default size 2x1, placed at first free position. Label auto-fills, user-editable.
- Edit modal: label only (feed_id immutable).
- Caddyfile: added `/news/*` to reverse proxy.

## Phase 5: Documentation Restructuring (2026-04-29)

- Split `HISTORY.md`, `PLAN_future.md`, `API.md` into focused sub-files (<150 lines each) under `docs/{history,plan,api}/`.
- Top-level files replaced with index tables containing a "When to read" column — keyword hints so agents only load relevant files.
- Added "Documentation conventions" section to `CLAUDE.md` defining the method: index-first navigation, keyword hints, separation of concerns (CLAUDE.md = current state, plan = WHY, history = WHAT/WHEN, api = HTTP contract).

### State at v4 (Phase A)

- 9 tables: added `news_feeds`, `news_articles`.

## Phase B: AI-Powered News (Gemini) — 2026-04-30

### Phase B1: Foundation

- **Models**: `ArticleCluster` table (label, summary, topic, article_count, expires_at). `NewsArticle` gains `cluster_id` (FK) and `duplicate_of_id` (self-FK).
- **Migrations**: `ee3945059e8c` (initial AI columns), `64c6ba48a9aa` (move summary to cluster, drop per-article summary).
- **Config**: `GEMINI_API_KEY` in `Settings` (empty = AI disabled).
- **AI service** (`app/services/ai.py`): `cluster_articles` groups headlines by event and generates short one-liner summaries. Uses `google-genai` (Gemini 2.5 Flash). Rate-limited at 4s between calls.
- **Dep**: `google-genai ^1.74.0`, httpx bumped to `^0.28.0`.

### Phase B2: Widget Modes + API

- **Widget modes**: `config.mode` discriminant — `single` (default), `topic`, `overall`.
- **API**: `GET /news/ai-status`, `GET /news/topics`, `GET /news/articles/topic?topic=X`, `GET /news/articles/clustered`. Topic and clustered endpoints both return `ClusteredArticleSchema[]`.
- **Widget validation**: topic mode activates all feeds for that topic, overall activates all feeds.
- **Orphan cleanup**: updated to respect topic/overall widgets.
- **Scheduler**: `scheduled_ai_pipeline` runs every 65 min, 5-min startup delay.

### Phase B3: Briefing UI

- **Frontend**: topic/overall widgets show briefing format — event one-liner + source badges (BBC, CNN, NYT) linking to original articles. Single-feed widgets unchanged (headline list).
- **AddWidgetModal**: mode selector (Single/Topic/Overall), topic picker.
- No per-article summaries — only cluster-level one-liners.

### State at v4B

- 10 tables: added `article_clusters` (with `summary` column).
- 2 new columns on `news_articles`: `cluster_id`, `duplicate_of_id`.
- 3 news widget modes: single (headline list), topic (briefing), overall (briefing).

## Phase C: AI Price Summaries + Pipeline Improvements — 2026-05-04

### Price Summaries

- **`PriceSnapshot.summary`** + `summary_updated_at` columns: one-liner explaining weekly price movement per asset.
- **Prompt**: uses sparkline % change + 30 recent headlines as context. Under 20 words. References specific events.
- **Migration**: added `summary` (text) and `summary_updated_at` (timestamptz) to `price_snapshots`.

### Pipeline Queue System

- **Problem**: Old system tried all assets sequentially per run. Gemini free-tier rate limit (15 RPM, 1,500 RPD) caused partial failures — only first few assets got summaries.
- **Fix**: Priority queue with no batch cap. Processes all eligible assets in order: no-summary first → oldest summary → skip fresh (<1hr). Stops immediately on 429/RESOURCE_EXHAUSTED instead of burning through doomed calls.
- **Rate limit propagation**: If clustering hits 429, entire run aborts (no wasted calls on summaries).
- **Delay**: Bumped from 4s to 5s (~12 RPM, safe margin under 15 RPM cap).
- **Interval**: Reduced from 65 min to 30 min so the queue drains faster.

### Capacity

- Free tier supports up to ~60 assets with hourly refresh.
- With 7 assets (current prod): 35 seconds per run, ~192 RPD — well within limits.

### Model Upgrade: Gemini 3 Flash

- **Previous**: Gemini 2.5 Flash — cheapest but weaker causal reasoning.
- **New**: Gemini 3 Flash (`gemini-3-flash-preview`) — better at connecting news events to price movements. Cost difference negligible (~$0.15/mo vs $0.05/mo at 7 assets).
- **Grounding (Google Search) deferred**: Available ($14/1K searches, 5K free/month) but not needed yet. Current approach uses cached headlines as context. Will add when service scales and revenue justifies it.
- **Paid tier**: Moving to paid Tier 1 (2,000 RPM, 10,000 RPD). Monthly budget: 10,000 KRW (~$7.50), supports up to 60 assets easily.

### State at v4C

- `price_snapshots` gains `summary`, `summary_updated_at`.
- Pipeline: 30-min interval, priority queue, rate-limit-aware abort.
- Model: Gemini 3 Flash (paid tier).
