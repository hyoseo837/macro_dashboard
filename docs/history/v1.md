# v1 — Price Dashboard

## 2026-04-20: Initial Implementation

- Analyzed `README.md`, `API.md`, and `PLAN.md` to define v1 scope (Prices only, News deferred).
- Visual North Star: `sample_design.html` (IBM Plex Mono/Sans, dark mode, high density).

### Backend (FastAPI)
- Docker + PostgreSQL 16 on port `5433`. SQLAlchemy 2.x async + Alembic.
- `yfinance` service for live prices + 30-day history. `APScheduler` refreshes every 15 min.
- Endpoints: `GET /assets`, `GET /prices`.

### Frontend (React + Vite)
- Vite, TypeScript, Tailwind CSS, TanStack React Query, Recharts.
- Polling-based state management (60s interval). Components: `AssetCard`, `Sparkline`, `ChangeBadge`.

### Design Iterations
- Sparkline: Recharts → hand-rolled SVG → back to Recharts (interactive hover/tooltips).
- Layout: 320px+ cards, graph attached to price section, enlarged change indicators.

## 2026-04-22: Asset Management & UX

- **Add**: Modal with Yahoo Finance autocomplete (`GET /assets/search`), auto-fills name/category/currency.
- **Delete**: 3-dot menu, fade-out animation, `DELETE /assets/{id}` cascades to `price_snapshots`.
- **Rename**: Inline editable input, `PATCH /assets/{id}`.
- Fixed `fetch_price_data` blocking event loop — moved to thread pool via `asyncio.to_thread`.
- Fast polling: 3s after adding asset until price appears, then reverts to 60s.

## 2026-04-23: v1.1.0 — Cleanup & v2 Prep

- Removed unused Vite template assets. Fixed favicon path.
- Bumped to `v1.1.0`. Adopted semver convention.
- v2 planning decisions made: widget-type picker, orphan cleanup, 6-col grid, clean slate.

### State at v1.1.0
- Backend on `:8000`, frontend on `:5173`. User-managed assets only.
- 8 endpoints: assets CRUD, search, currency, prices, health.
