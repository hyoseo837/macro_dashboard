# v1 & v2 Plans (Complete)

## v1 — Single-User Price Dashboard ✅

- Asset cards with live prices, sparklines, change badges
- Add/delete/rename assets via Yahoo Finance autocomplete
- Backend scheduler refreshes prices every 15 min, frontend polls every 60s
- Single `assets` + `price_snapshots` tables, no user accounts

## v2 — Widget System & Customizable Grid ✅

Replaces the asset grid with a general-purpose widget system.

### Design Decisions

1. Orphan asset cleanup on widget delete
2. Display name is widget-owned (`config.label`)
3. Default size 1x1, grid 6 cols, no compaction
4. Clean slate (no v1 data migration)

### Widget Types

- **Asset**: price card with sparkline, responsive at 1x1/2x1/1x2/2x2
- **Time**: live clock (analog/digital), responsive at all sizes

### Database

- `widgets`: `id`, `type` (enum), `config` (jsonb), `layout_x/y/w/h`
- `assets` + `price_snapshots` remain as shared cache

### Phases (all complete)

1. Backend: Widget model + CRUD + orphan cleanup ✅
2. Frontend: Types + API + hook ✅
3. Frontend: Widget grid + edit mode ✅
4. Asset widget: responsive price cards ✅
5. Time widget: live clocks ✅
6. Add widget modal: two-step flow ✅
