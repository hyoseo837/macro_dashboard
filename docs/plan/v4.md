# v4 Plan — News Widgets (In Progress)

Two-phase approach: Phase A (headlines) ships first. Phase B (AI) adds Gemini.

## Design Decisions

1. **Two phases**: Headlines-only (RSS) first, AI features second.
2. **Single-feed widgets**: Each news widget = one RSS feed. 1:1 like asset widgets.
3. **Feed catalog**: Predefined in code (22 feeds). Not user-editable.
4. **On-demand activation**: Feeds only fetched when a widget references them.
5. **Hourly refresh**: Articles older than 7 days cleaned up daily.
6. **Minimum size**: 2x1. Article count scales with size automatically.
7. **Click to read**: Headlines open original article in new tab.
8. **Label**: Auto-generated from feed name, user-editable.
9. **No custom RSS URLs**: Catalog only.

## Phase B — AI-Powered News (Gemini)

**Implemented.** AI event clustering with briefing summaries using Gemini 2.5 Flash (`google-genai`).

**Design**: The AI groups headlines from all sources into events and writes a short one-liner per event (e.g. "Oil prices surge as US strikes Iranian refineries"). Topic/overall widgets show this briefing format with source badges linking to original articles. Single-feed widgets remain unchanged (headline list).

**No per-article summaries** — the briefing one-liner per event cluster is the only AI-generated text. Simpler, more useful, fewer API calls.

**Widget config modes** (same `news` WidgetType, discriminated by `config.mode`):
- `single` (default, backward-compat): `{feed_id, label}` — headline list
- `topic`: `{mode: "topic", topic: "technology", label: "All Tech News"}` — briefing format
- `overall`: `{mode: "overall", label: "All News"}` — briefing format

**Scheduler**: AI pipeline runs every 65 min (offset from 60-min feed refresh), 5-min delay on startup.
**Graceful degradation**: all features disabled when `GEMINI_API_KEY` is empty string.
**Deps**: `google-genai`, `GEMINI_API_KEY` env var.
