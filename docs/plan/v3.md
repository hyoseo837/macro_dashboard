# v3 Plan — Multi-User (Complete)

## Design Decisions

1. Email/password only (no OAuth). JWT stateless. Invite-only registration.
2. Password policy: min 8 chars, no complexity requirements.
3. Admin creates invite codes. Open registration deferred.
4. Admin panel: invite code CRUD + user list. No role hierarchy.
5. Clean slate — no v2 widget migration. Default widgets on registration.
6. Password reset via SMTP from the start.
7. Landing page for unauthenticated visitors. `react-router-dom` routing.
8. Assets remain global. User-scoping at widget level only.
9. Access token in memory, refresh token as `httpOnly` cookie.
10. First admin via CLI (`create-admin`).

## Auth System

- Access token: 30 min, stateless, payload: `user_id` + `is_admin`
- Refresh token: 7 days, DB-stored hash, rotated on refresh
- Password hashing: `bcrypt` directly (passlib incompatible with bcrypt 5.x)

## Database (new tables)

- `users`: email, hashed_password, birth_date, is_admin, timestamps
- `invite_codes`: code, created_by, max_uses, use_count, expires_at
- `refresh_tokens`: user_id, token_hash, expires_at, revoked
- `password_reset_tokens`: user_id, token_hash, expires_at, used
- `widgets`: added `user_id` FK (NOT NULL)
